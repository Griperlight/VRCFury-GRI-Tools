# ⚡ Gri Tools — VRCFury Menu Builder

A Unity Editor tool that simplifies the creation and management of VRChat avatar menus using VRCFury. Built for real avatar workflows — less clicking, less manual setup, more time animating.

## What is this?

Setting up avatar menus in VRChat normally means configuring parameters, expression menus, expression parameters, animations, and then wiring everything together in VRCFury by hand. This process gets tedious fast when an avatar has dozens of toggles organized across multiple submenus.

**Gri Tools — VRCFury Menu Builder** is an Editor window that automates that process. Select your avatar, define your toggles through a visual interface, and the tool handles the rest: it builds the correct VRCFury structure, assigns parameters with consistent naming, and organizes submenus without any manual steps.

---

## Architecture

The tool follows a deliberate, clean architecture:

```
Avatar Root
├── VRCFury Component       ← Contains only FixWriteDefaults (Auto)
└── Menus/
    └── VRCFury Component   ← Contains all Toggle features
```

This separation keeps the root component clean and groups all menu logic into a single predictable location.

## Current Features

### Toggles
- Create toggles with a name, icon, and one or more assigned GameObjects
- Per-object mode: **Turn On** or **Turn Off** independently per slot
- Options: Default ON, Inverted, Saved Parameter, Use Int, Slider/Radial
- Auto-generated parameter with VRChat-safe naming (`Toggle_ObjectName`)
- Edit or delete existing toggles directly from the window
- Real-time validation before saving

### Submenus
- Create submenus with nested hierarchy (e.g. `Clothing/Casual/Tops`)
- Tree view with foldouts
- Quick button to create a toggle directly inside a submenu
- Recursive deletion of submenus and their children

### Parameters
- View all parameters generated by existing toggles
- Memory usage estimate in bits (out of the 256 available)
- Visual usage indicator: green / yellow / red
- Duplicate detection with a warning on create

### Automatic Setup
- Avatar auto-detection from the Hierarchy (auto-selects on click)
- Single button to configure everything: creates the root VRCFury with FixWriteDefaults and the `Menus` child object with its own VRCFury component
- Status bar that reflects the real state of the avatar at all times
- Full Undo support on every operation

## Installation

1. Clone or copy the files into any folder inside `Assets/Editor/` in your Unity project
2. Make sure VRCFury is installed via VCC
3. In Unity: **Gri Tools → VRCFury Menu Builder**

```
Assets/
└── Editor/
    └── GriTools/
        ├── VRCFuryMenuBuilderWindow.cs
        ├── VRCFuryBridge.cs
        ├── MenuBuilderStyles.cs
        └── MenuBuilderData.cs
```

## Code Structure

| File | Responsibility |
|---|---|
| `VRCFuryMenuBuilderWindow.cs` | Main EditorWindow — all UI and interaction logic |
| `VRCFuryBridge.cs` | Reflection layer — communicates with VRCFury internals without a direct assembly dependency |
| `MenuBuilderStyles.cs` | UI styles and color palette |
| `MenuBuilderData.cs` | Data models: `ToggleData`, `SubmenuData`, `ObjectToggleEntry` |

The bridge uses reflection to avoid a hard reference to the VRCFury assembly, making the tool more resilient to version changes.

---

## Design Notes

- **No hardcoded dependencies** — the bridge resolves VRCFury types at runtime, not at compile time
- **Full Undo support** — every operation registers an undo step so changes can be reverted from Unity
- **SerializedProperty for writes** — toggles are written via `SerializedObject` to correctly handle Unity 2022's `[SerializeReference]` system
- **Reflection + SerializedProperty for reads** — combined approach to read the actual serialized state rather than a stale reflection snapshot
